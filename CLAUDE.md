# Beauty Tech Japan プロジェクト固有ルール

## 画像パフォーマンス（自律対応）
- `unoptimized: true` は絶対に設定しない。Vercelの画像最適化を常に有効にする
- `next/image` の `sizes` プロップは実際の表示サイズに合わせて正確に設定する
- 外部画像ホスト（Unsplash等）は `remotePatterns` に必ず登録する
- `formats: ["image/avif", "image/webp"]` を常に有効にする
- ファーストビューの画像には `priority` を付ける
- 作業の中で画像パフォーマンス問題を発見したら、ユーザーに言われる前に自分で修正する

## UI/UX（自律対応）
- **PCとスマホで臨機応変にレイアウトを整える**（ユーザーに言われなくても自分で判断して対応する）
- スマホ（375px）・PC（1280px）両方で見た目を常に意識する
- PC では情報を横並びで広く使い、スマホでは縦積み・大きいタップ領域で見やすくする
- 横スクロールが発生しないよう `overflow-hidden` や `flex-wrap` を適切に使う
- フォーム・ボタンはスマホで切れないよう `flex-col sm:flex-row` パターンを使う
- カテゴリフィルターなど多数の要素は `flex-wrap` で折り返しにする
- グリッドはスマホで `grid-cols-1` または `grid-cols-2`、PCで `grid-cols-3` 以上に調整する
- テキストサイズはスマホで読みやすく（最小 `text-xs`）、PCで適切に大きくする
- 作業の中でUI/UXの問題を発見したら、ユーザーに言われる前に自分で修正してデプロイする
- フロントエンドを変更したらデプロイ前に Playwright でスクリーンショットを確認する

## 自律ブラッシュアップ
- 作業の合間にサイト全体を見渡し、改善できる点があれば自分で実施する
- 特に以下を定期的にチェックして改善する：
  - 画像の読み込み速度・最適化設定
  - モバイルレイアウトの崩れ
  - 誤ったラベル・テキスト（例：「人気順」が実は「古い順」など）
  - リンク切れ・未実装の機能（ボタンはあるがページがないなど）
  - About/説明ページの内容が実態と合っているか

## デプロイ
- 作業完了後は必ず `vercel --prod` でデプロイする（git push だけでなく）
